# AI Usage Policy

This project was largely built with AI assistance (Claude Code), and AI
tools remain welcome for contributions. That said, quality matters more
than how the code was produced.

## Rules for Contributions

- **All AI usage must be disclosed.** State the tool you used (e.g.
  Claude Code, Cursor, Copilot) and the extent of AI assistance in
  your pull request description.

- **You must fully understand the code you submit.** If you can't
  explain what your changes do and how they interact with the rest
  of the project without the aid of AI tools, don't submit them.

- **Issues and discussions can use AI assistance but must have a full
  human-in-the-loop.** Any AI-generated content must be reviewed and
  edited by a human before submission. AI tends to be overly verbose —
  trim it down to what matters.

- **No AI-generated media** (art, images, videos, audio). Text and
  code are the only acceptable AI-generated content, per the other
  rules in this policy.

## There are Humans Here

Every issue and pull request is read by humans. It is disrespectful
to submit low-effort, unreviewed AI output that puts the burden of
validation on maintainers. AI can produce great work, but only when
the person driving it takes responsibility for the result.

## AI is Welcome Here

This project was created with significant AI assistance, and we see AI
as a productive tool. **This policy is not anti-AI** — it exists to
ensure that contributions meet a quality bar regardless of how they
were produced.

_This policy is inspired by [Ghostty's AI Policy](https://github.com/ghostty-org/ghostty/blob/main/AI_POLICY.md)._
